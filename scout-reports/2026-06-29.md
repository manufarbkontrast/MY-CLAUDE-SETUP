- **47% weniger Tokens**
- **16% geringere API-Kosten**
- Getestet über 7 reale Codebases

### Installation
```bash
npm i -g @colbymchenry/codegraph
codegraph install    # Auto-Detection: Claude Code, Cursor, Codex etc.
codegraph init       # Baut Index im Projektverzeichnis (.codegraph/)
```

### Überschneidung mit bestehendem Setup
**Keine.** Das Setup hat aktuell keine Code-Indexierung. codegraph ergänzt die bestehenden Performance-Rules (`performance.md`) ideal — die Rules empfehlen Token-Sparsamkeit, codegraph liefert die Infrastruktur dafür.

### Risiko
Gering. 100% lokal, kein Cloud-Dienst. Index wird bei Code-Änderungen automatisch aktualisiert. Einziger Nachteil: zusätzlicher Disk-Space für `.codegraph/`-Verzeichnis pro Projekt.

## 6. OpenBB — Finanzdaten als MCP Server

| | |
|---|---|
| **URL** | https://github.com/OpenBB-finance/OpenBB |
| **Sterne** | 69.819 ⭐ |
| **Kategorie** | MCP Server / Finanz-Tool |
| **Relevanz** | **Hoch** — MCP-native Finanzdaten |
| **Priorität** | **#6 — Evaluieren, wenn Finanzdaten benötigt** |

### MCP-Integration
OpenBB bietet einen offiziellen MCP Server (`openbb-mcp-server`), der die gesamte Finanzdaten-API über MCP exponiert.

### Setup für Claude Code
```bash
pip install openbb openbb-mcp-server
openbb-mcp-server --default-categories equity,economy,crypto

# In Claude Code:
claude mcp add --transport http openbb-mcp http://localhost:8001/mcp/
```

Konfiguration über `~/.openbb_platform/mcp_settings.json`.

### Verfügbare Daten
- Aktien (Fundamentals, Technicals, Historische Kurse)
- Crypto (Preise, On-Chain-Metriken)
- Wirtschaftsdaten (Makro-Indikatoren, Zinssätze)
- Devisen, Futures, Optionen
- SEC Filings, Insider Trading
- ETFs, Mutual Funds

### Synergie mit bestehendem Setup
Kombiniert mit **dbhub** (bereits installiert) ergibt sich ein mächtiger Stack:
- OpenBB → Finanzdaten abrufen
- dbhub → In lokale DB speichern/abfragen
- Claude → Analyse, Visualisierung, Reporting

### Kosten
Open Source (AGPLv3). OpenBB Workspace (UI) ist kostenpflichtig, aber die API/MCP-Server sind frei nutzbar.

## 7. TradingAgents — Multi-Agent-Architektur für Finanzanalyse

| | |
|---|---|
| **URL** | https://github.com/TauricResearch/TradingAgents |
| **Sterne** | 89.582 ⭐ |
| **Kategorie** | Finanz-Tool / Architektur-Inspiration |
| **Relevanz** | **Mittel-Hoch** — Architektur-Pattern übertragbar |
| **Priorität** | **#7 — Als Architektur-Referenz** |

### Agentenrollen (übertragbar auf andere Domains)

```
┌─────────────────────────────────────────┐
│            Fund Manager                  │
│         (Finale Entscheidung)           │
├──────────┬──────────┬───────────────────┤
│ Portfolio │  Risk    │    Trader         │
│ Manager   │ Manager  │    Agent          │
├──────────┴──────────┴───────────────────┤
│   Bullish Researcher │ Bearish Researcher│
│       (Pro-Debatte)  │ (Contra-Debatte)  │
├──────────────────────┴──────────────────┤
│ Fundamental │ Sentiment │ News │ Technical│
│  Analyst    │  Analyst  │Analyst│ Analyst │
└─────────────┴───────────┴──────┴─────────┘
```

### Übertragbare Patterns
1. **Adversarial Debate**: Bullish vs. Bearish Researcher — anwendbar auf jede Entscheidungsfindung
2. **Spezialisierte Analysten**: Jeder Agent hat eine Datenquelle/Perspektive
3. **Hierarchische Aggregation**: Analyst → Researcher → Manager → Final Decision
4. **LLM-Agnostik**: Unterstützt Claude, GPT, Gemini, lokale Modelle via Ollama

### Empfehlung
Nicht direkt installieren (Trading-spezifisch), aber die Multi-Agent-Debate-Architektur als Pattern für eigene Analyse-Workflows übernehmen — z.B. für SEO-Audits (Technisch vs. Content vs. UX-Perspektive) oder Business-Case-Bewertungen.
